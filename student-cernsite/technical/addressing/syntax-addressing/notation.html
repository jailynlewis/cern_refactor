Notation

This section explains the notational conventions used in WWW specifications, especially an augmented Backus-Naur Form (BNF).

Notational Conventions (BNF)

  1. Rule Naming: Use the rule name directly, e.g., RuleName. Literal text is in quotes.
  2. Alternatives: Use | for alternatives, e.g., "foo" | "bar". Avoid slashes "/" because they are used in paths.
  3. Repetition: * indicates repetition. 1*element means at least one occurrence.
  4. Specific Repetition: Use <n>(element) for exactly n occurrences. Example: 2DIGIT.
  5. Lists: Use # for comma-separated lists. Example: 1#element.
  6. Comments: Start a comment with ; for the rest of the line.

Changes and Additions for WWW

  • Non-terminals are PascalCase, e.g., NonTerminalName.
  • Rules are written as: NonTerminal ::= RuleDefinition.
  • C-style escape sequences: \n (newline), \r (carriage return), \t (tab), \b (backspace), \f (form feed).
  • Avoid angle brackets to prevent conflicts with HTML tags.

General Rules

HTTP data is line-based. Strings are ASCII in double-quotes. Escape internal quotes with \".

Example

Method ::= "GET" | "POST"
URL    ::= "http://" Host [ "/" Path ]
Host   ::= 1*( ALPHA | DIGIT | "." | "-" )
Path   ::= *( ALPHA | DIGIT | "/" | "-" | "_" )
