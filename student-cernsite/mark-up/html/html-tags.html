HTML Tags

This is a list of tags used in the HTML language. Each tag starts with a tag opener (<) and ends with a tag closer (>). Many tags have corresponding closing tags which are identical except for a slash after the tag opener.

Some tags take parameters, called attributes. Certain attributes are followed by a value. See the Anchor tag, for example. The names of tags and attributes are not case sensitive.

Currently HTML documents are transmitted without normal SGML framing tags. See specifications elsewhere.

Title

The title of a document is given between <TITLE> tags.

There should only be one title in any node. It identifies the content of the node in a wide context.

Base Address

Anchors specify addresses relative to the current document. If a document has multiple addresses, the browser needs a base address.

NOT CURRENTLY USED.

Anchors

The format of an anchor is:

<A HREF="...">Link</A>
HREF
Defines the destination of a link. See network address format. Can also reference a fragment using #identifier.
NAME
Defines a destination anchor using the value after the hash sign.
TYPE
Defines the relationship of the link. Optional.

If the anchor refers to another document, it uses a relative name.

IsIndex

Marks the document as searchable. Queries are added using a question mark and keywords.

Plaintext

Indicates that all following text is literal. Often used for fixed-width formatting. 


12345678
! ! ! !
Column: First Second
Index: 1 2

Paragraph

Indicates a new paragraph. Representation depends on styles.

Headings

HTML supports multiple heading levels from <H1> to <H6>. H1 is the highest level.

Address

Used for address information or signatures.

Highlighting

Highlighted phrases may be nested. NOT CURRENTLY USED.

Lists

A list is a sequence of paragraphs.

  • UL — unordered list
  • MENU — compact list
  • DIR — directory-style list

These definitions originate from the AAP tag set.
