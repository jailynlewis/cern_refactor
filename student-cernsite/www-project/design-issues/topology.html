Topology

Here are a few questions about the underlying connectivity of a hypertext web.

Are links two- or multi-ended?

The term "link" normally indicates two ends. Variations include links with multiple sources or multiple destinations. Constructs relating more than two anchors map onto logic systems like predicate calculus. For example, the "Aquanet" system from Xerox PARC (paper at HT91). Here we focus on conventional links with two ends.

Should the links be monodirectional or bidirectional?

Bidirectional links exist in both directions. Advantage: adds information automatically (e.g., if A includes B, B is part of A). Disadvantage: may constrain the author. KMS and HyperCard have one-way links; Enquire has two-way links. Reverse links can also help manage dangling links in a database.

A compromise is to make links one-way in the data model but create a reverse link automatically when allowed. See Building Back-links.

Should anchors have more than one link?

One anchor leading to multiple destinations can be ambiguous for the user. A many-to-one mapping is suggested, though JFG prefers showing a small menu if a link is ambiguous. Microcosm implements this.

Should links be typed?

Typed links carry semantic information to manage data more efficiently. Default type "untyped" exists if types are implemented. See a list of some types. Links can help with graphical overviews and automatic tracing.

Link typing should not allow multiple types per link; instead, use separate links.

Should links contain ancillary information?

Ancillary information includes date, version, authorship, comments. Including this on the link itself can complicate the data model. Better to store in the source node or intermediate annotation node.

Should a link contain Preview information?

Preview allows the reader to see the target before following the link. This is optional and usually handled via caching or descriptive highlights. Fast graphic overviews may make a preview unnecessary.

Other relevant terms: Private overlaid web, Generic Links.
